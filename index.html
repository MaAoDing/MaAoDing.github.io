忒修斯之船

发表于 2022-04-21 本文字数： 392 阅读时长 ≈ 1 分钟

一艘船，在大海上航行多年，修修补补，几乎全身零件都已经更换，现在它身上只有少数出厂时的材料。你认为，这艘船还是原本的船吗?

进阶
同样的，人体细胞不断更新，你身体里的大部分器官早已重组了，你的身体早就不是当初那个身体了，你还认为你一直是同一个你吗？
你不是过去，因为过去是Ta所创造的。
你不是现在，因为Ta所创造的过去决定现在的你。
你不是未来，因为未来由Ta掌管。

所以，你甚至无法干涉你自己的行动，那么你又能做到什么？
是在命运的牢笼里作茧自缚，还是顺从历史的洪流奔涌而下？
是用努力将这搅不破的蛛网搅破，还是任凭这绝望的世界将你碾成粉末？
网由点连接而成，每一点之间的连线让点们不能挣扎分毫，但网也由此稳定。
你是网中的一点，是这稳定的网中无法挪动的一点！
你是要为网的稳定献出自己的自由，亦或是任凭满身伤痕也要离开这网？
十分抱歉，你没得选，你只能成为网的一部分，而不是一个自由的点。
你能做到什么？

无数次的BE的失败，只是为了最后的HE的胜利。
在这无数的BE里，拯救世界的英雄失败了，但是英雄知道，他还会再回来，通过无数次死亡，去换取一次胜利。
但在这个过程中最后只有英雄和HE的人们到达了梦想乡。

掌握复活技能后的英雄，哪怕内心再坚定，在这永生的魅力下，也会逐渐丧失自己的初衷，甚至堕入黑暗，替代魔头成为了世界BOSS，霸占世界，成为永生不死的魔王。
拯救世界？开什么玩笑？魔王会把自己从王座上拽下来吗？然后一脸忏悔地说我有罪？
无限次的复活，使得英雄根本无法战胜，每一次的谋略，最终只会成为朝向自己的利矛。
但这次不一样，我们要让英雄没有下次！所有来自BE的人，他们早已放弃希望，支撑他们或者来到这里的目标，只有一个，那就是，真正地杀死英雄！

Windows11安卓子系统安装教程

发表于 2022-03-12 更新于 2022-04-21 本文字数： 564 阅读时长 ≈ 1 分钟

第一步 将你的地区切换至美国

1.打开设置-时间和语言-语言和区域
2.将国家和地区切换为美国

第二步 安装亚马逊商店

1.打开微软商店搜索并下载Amazon Appstore

第三步 重启

第四步 安装apk安装器

1.微软商店下载任意apk安装器

第五步 开启子系统开发人员选项

1.开始里找到适用于Android的Windows子系统设置
2.开启开发人员选项

第六步 安装软件

1.打开先前下载的apk安装器
2.安装准备好的安装包（这里推荐安装酷安，后续软件下载及更新会更方便）

结语

对微软向移动端的妥协，我本身是感到悲哀的，但同时，这个子系统也为我提供了极大的便利。
一些没有电脑端或电脑端是被阉割的软件，也可以通过子系统来实现运行直接跑手机版。
在这个移动设备为主要电子产品的时代，无疑PC正在逐渐走向小众。
以前，手机只不过是一个通讯设备。
现在，手机开始拥有各种各样的功能，甚至有了PC没有的功能。
未来，手机将会集成PC的所有的功能，没有的功能，将PC彻底赶出大众的视野。
PC的落幕或许并不是一件坏事，它代表人类的科技更进一步，信息时代的下一个纪元。
未来，或许会出现更多的比手机更加优秀的设备，它们也将取代手机。
哪怕未来真的出现了植入人体的高度集成化电子设备，实现真正的人机结合，我仍会铭记所属于信息时代的开端。

PS：拥有触屏功能的屏幕可以更为和谐地体验子系统

Hexo博客搭建教程

发表于 2022-03-06 更新于 2022-07-02 本文字数： 609 阅读时长 ≈ 1 分钟

第一步 安装Node.js环境

1.node.js官网链接”https://nodejs.org/zh-cn/"

第二步 安装hexo框架

1.在命令行里输入npm install hexo-cli -g

第三步 初始化博客

1.依次在命令行里输入1 hexo init blog
2 cd blog
3 npm install

第四步 启动本地博客服务（本地进行实验）

1.在命令行里输入hexo s

第五步 部署到Github

1.Github创建一个新的仓库 YourGithubName.github.io
2.npm install hexo-deployer-git –save 在blog目录下安装git部署插件
git config –global user.email xxx
git config –global user.name “xxx”配置git
3.配置_config.yml
# Deployment
## Docs: “https://hexo.io/docs/"
deploy:
type: git
repo: “https://github.com/YourGithubName/YourGithubName.github.io.git/"
branch: master
4.命令行输入hexo clean以及hexo g
5.部署到Github仓库里hexo d
6.访问这个地址可以查看博客”https://YourGithubName.github.io/"
